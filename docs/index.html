Server Next

A library to easily create a modern JS server. Handles HTTP, Websockets and all the small details: 

npm install @server/next
Getting Started Documentation 
import server from "@server/next";

export default server()
  .get("/", () => "Hello world")
  .post("/users", ctx => {
    console.log(ctx.body);
    return { id: 100 };
  });

💬 Just return a value

Return a string, an object, a status code, a file or a stream, and Server.js sends the right response with the right headers. No res.send() boilerplate. 

import server, { file } from "@server/next";

server()
  .get("/text", () => "Hello")         // text/plain
  .get("/json", () => ({ ok: true }))  // application/json
  .get("/gone", () => 404)             // status code
  .get("/file", () => file("./a.pdf")); // streamed file

✨ Fully Typed

Advanced type inference so you never mistype a URL parameter, a body field, or a provider.

Every route method carries a description, a representative example and even a link to the docs, right in your editor.


🔑 Authentication built in

Add OAuth (GitHub, Google, Microsoft, Discord, Facebook, Apple) or email and password with a single option.

Sessions, signed cookies and CSRF-protected flows are handled for you, and the logged-in user is on ctx.user for every request. 

import server from "@server/next";

export default server({ store, auth: "cookie:github" })
  .get("/me", ctx => ctx.user || 401);
// send users to /auth/login/github to sign in

📦 Uploads that just work

Files in a request are streamed straight to storage as they arrive, never buffered in memory, so even large uploads stay cheap.

Point uploads at a local folder or an S3-style bucket, and read the stored file right off ctx.body. 

export default server({ uploads: "./uploads" })
  .post("/avatar", ctx => {
    // { name, id, path, type, size }
    return ctx.body.avatar;
  });

🔌 Real-time WebSockets

Handle WebSocket events with the same routing style. Reply to one client with ctx.socket, or broadcast to all of ctx.sockets. 

server()
  .get("/", () => file("./index.html"))
  .socket("message", ctx => {
    ctx.socket.send(`You said: ${ctx.body}`);
  });

🌍 Write once, run anywhere

The exact same file runs on Node, Bun, Cloudflare Workers and Netlify Functions. No adapters, no platform-specific entry points. 

import server from "@server/next";

// Deploys unchanged to Node, Bun, Cloudflare, Netlify
export default server()
  .get("/", () => "Hello from anywhere");

🧪 Testing without the ceremony

Every server has a built-in, in-process test client. No ports, no mocking, no network round-trips.

Call your routes directly and assert on the real Response, so tests stay fast and close to production. 

const api = server()
  .get("/hello", () => "Hi!")
  .test();

const res = await api.get("/hello");
expect(await res.text()).toBe("Hi!");
